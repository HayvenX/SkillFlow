# Developer Guide (SkillFlow)

Це коротке зведення правил розробки для нашої команди. Дотримання цих кроків допоможе уникнути конфліктів у коді та плутанини з гілками.

## 1. Структура гілок

- **main** — стабільний код. Сюди зміни потрапляють лише з `dev`. Прямі пуші заборонені.
- **dev** — основна робоча гілка. Всі нові задачі починаються звідси.

## 2. Алгоритм роботи над задачею

Кожну нову фічу або баг робимо в окремій тимчасовій гілці:

1.  **Оновіть локальний dev:**
    `git switch dev` та `git pull`
2.  **Створіть гілку під задачу:**
    наприклад, `git branch fix/login-redirect-bug` (використовуйте зрозумілу коротку назву)
3.  **Перейдіть на нову гілку:**
    `git switch fix/login-redirect-bug`
4.  **Працюйте та робіть комміти.**
5.  **Відправте гілку на сервер:**
    `git push origin fix/login-redirect-bug`
6.  **Створіть Pull Request (PR):**
    - Зайдіть у репозиторій на GitHub.
    - Ви побачите жовту панель із вашою гілкою та кнопкою **"Compare & pull request"** (натисніть її).
    - Перевірте, щоб напрямок був: `base: dev` ← `compare: task.назва-задачі`.
    - Натисніть **"Create pull request"**.

## 3. Після мержа (Прибирання)

⚠️ **ВАЖЛИВО:** Видаляйте тимчасову гілку ТІЛЬКИ ПІСЛЯ ТОГО, як статус Pull Request на GitHub змінився на **Merged** (фіолетова іконка).

1.  Перейдіть локально на dev: `git switch dev`
2.  Оновіться (щоб підтягнути замержені зміни): `git pull`
3.  Тепер можна безпечно видалити тимчасову гілку: `git branch -d task.назва-задачі`

## 4. Поради та самоперевірка

- **Не "висіть" на гілці main.** Використовуйте її лише як еталон.
- **Перевірити поточну гілку:** `git branch --show-current`.
- **Статус змін:** `git status` (робіть перед кожним перемиканням гілок).
- **Перед новою задачею:** Завжди підтягуйте актуальний стан `dev`.
- **Конфлікти:** Якщо виникають складні конфлікти — пишіть у чат, розберемось разом.

## 5. Синхронізація (якщо main випередив dev)

Якщо код випадково потрапив у `main` в обхід `dev`, наздоганяємо так:

1. Створіть PR на GitHub: **base: dev** <— **compare: main**.
2. Якщо бачите зелений напис **"Able to merge"** — завершуйте мерж.
3. Якщо є конфлікти — натисніть **Resolve conflicts**, видаліть маркери (`<<<<`, `====`, `>>>>`) у браузері, натисніть **Mark as resolved** і завершіть мерж.
4. Локально на комп'ютері обов'язково зробіть: `git switch dev` та `git pull`.
